Hearst Patterns
Hearst Patterns can be defined as lexical-syntactic patterns. Here we use them to extract our n-triples by using them as patterns. Since we define them as regex patterns, our main word to be finding should be PREDICATE. This can be present in either of the two forms:-
  • The specific predicate itself or
  • An easier way - verb+preposition

    • This can then be mapped to a DBpedia property, while annotating. The second parameter is the first/last which denotes if the before word captured is the SUBJECT or the last one.
    Parse-Tree
    Nothing which can be configured here.
    Dependencies
    The Dependency level dettermines how close two words (SUBJECT-OBJECT) are related, through the PREDICATE. The higher the level, the lesser confidence.
    The minimum which can be set is 1, which would you provide you with hypernyms. 2 or more will provide you with Depdency triples. The default value being used is 2.
