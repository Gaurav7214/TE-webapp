Evaluations

Here we explain the working and logic behind DBpedia Extract. Let's take the following text as an example :-

Beyoncé Giselle Knowles-Carter is an American singer, songwriter, record producer and actress. Born and raised in Houston, Texas, she performed in various singing and dancing competitions as a child, and rose to fame in the late 1990s as lead singer of R&B girl-group Destiny's Child. Managed by her father, Mathew Knowles, the group became one of the world's best-selling girl groups of all time. Their hiatus saw the release of Beyoncé's debut album, Dangerously in Love (2003), which established her as a solo artist worldwide, earned five Grammy Awards and featured the Billboard Hot 100 number-one singles "Crazy in Love" and "Baby Boy".

We will now use the above text to understand each of the methods used by the application, in other words we will look at various configurations of the application and see what results are being produced and how they are being produced.

Configuration - 1 : Hearst Patterns

The hearst patterns method allows users to add additional hearst patterns, but however the application list of hearst patterns which can be added. Another set of hearst patterns are created by using the DBpedia properties which are present. Running the following configuration we get the following triples

(<http://dbpedia.org/resource/Houston>, formOf, <http://dbpedia.org/resource/Child>)
(<http://dbpedia.org/resource/Late_Antiquity>, typeOf, <http://dbpedia.org/resource/Lead>)

Hearst patterns depend heavily on syntax of the sentence and thus their performance is dependent on the sentence and cannot be predicted. In the following case, the hearst patterns don't perform to the mark

Configuration - 2 : Using Parse Tree Method

The parse tree method can be switched on and off in the configuration. The parse tree method relies on an algorithm which is run the parse-tree generated by the parse on the sentence. The triples identified by setting this as the configuration are

(<http://dbpedia.org/resource/Houston>, <raised>, <http://dbpedia.org/resource/Houston>)
(<http://dbpedia.org/resource/Beyoncé>;, <typeOf>, <http://dbpedia.org/resource/Singing>)
(<hiatus>, <featured>, <http://dbpedia.org/resource/Synthesizer>)
(<father>, <Managed>, <father>)

The Parse Tree Method uses the generated parse tree to find triples (to be specific the treebank parse tree). Since this is independent of the syntax of the sentence (in a way), the results obtained by the followingm method will be better than the hearst pattern results, however the above results still can be further improved

Configuration - 3 : Using Dependencies

Dependencies can be switched on and off in the configuration. This method relies on the CoNLL-U dependencies which are obtained from the text. These dependencies are then checked for triples. The triples identified by setting this as the configuration are

(<http://dbpedia.org/resource/Singing>, <hypernym> (low confidence), <http://dbpedia.org/resource/Rhythm_and_blues>)
(<http://dbpedia.org/resource/Singing>, <http://purl.org/linguistics/gold/hypernym>, <http://dbpedia.org/resource/Giselle>)
(<http://dbpedia.org/resource/Synthesizer>, <hypernym> (low confidence), <http://xmlns.com/foaf/0.1/album<)
(<http://dbpedia.org/resource/Insanity>, <establish_as>, <http://dbpedia.org/resource/Artist>)
(<album>, <earn_in>, <http://dbpedia.org/resource/Insanity>)

The Dependencies Method uses the dependencies obtained from the text to determine triples. This is done by traversing the dependencies assuming it is a connected undirected graph. Using this method (better explained in the blog posts) the subject, onject along with the connecting verb can be extracted

Configuration - 4 : Using Dependencies with coreferences

Dependencies with coreferences can be switched on and off in the configuration. This method is an extension to the Dependenices, instead now the text coreferences are now utilised in extracting triples. The triples identified by setting this as the configuration are

(<http://dbpedia.org/resource/Singing>, <hypernym> (low confidence), <http://dbpedia.org/resource/Rhythm_and_blues>)
(<http://dbpedia.org/resource/Singing>, <http://purl.org/linguistics/gold/hypernym>, <http://dbpedia.org/resource/Giselle>)
(<http://dbpedia.org/resource/Synthesizer>, <hypernym> (low confidence), <http://xmlns.com/foaf/0.1/album<)
(<http://dbpedia.org/resource/Insanity>, <establish_as>, <http://dbpedia.org/resource/Artist>)
(<album>, <earn_in>, <http://dbpedia.org/resource/Insanity>)
(<http://dbpedia.org/resource/Beyoncé>, <perform_as>, <http://dbpedia.org/resource/Child>)

The Dependencies with coreferences Method will help better the results by adding coreferences. In our case, this method does help in adding one extra triple to the triples which have been extracted (namely the last one)
